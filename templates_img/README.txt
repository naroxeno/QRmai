QRmai OpenCV 模板图目录
========================

此目录存放用于 OpenCV 模板匹配的参考图像，用于自动识别 P1/P2 的屏幕位置。

## 模板图说明

| 文件名          | 用途                       | 来源     |
|----------------|---------------------------|----------|
| p1.png         | 微信「生成二维码」按钮的截图 | 开发者   |
| p2.png         | 微信二维码消息区域的截图     | 开发者   |
| p1_user.png    | 同上（用户自定义）          | 用户     |
| p2_user.png    | 同上（用户自定义）          | 用户     |

## 优先级

  1. p1_user.png / p2_user.png（用户模板，最高优先级）
  2. p1.png / p2.png（开发者模板）
  3. config.json 中的 p1/p2 硬编码坐标（fallback）

## 如何使用

1. 打开微信并进入舞萌/中二的二维码页面
2. 使用截图工具截取 P1 和 P2 对应的 UI 元素：
   - **P1**: 截取二维码页面中用于生成二维码的按钮
   - **P2**: 截取二维码消息区域的典型外观
3. 将截图保存为 p1_user.png / p2_user.png 放入此目录
4. 在 settings 页面或 config.json 中将 auto_detect_p1p2 设为 true

## 模板图要求

- 格式：PNG
- 内容：只包含目标 UI 元素，尽量裁剪到最小范围
- 建议尺寸：50-200 像素宽高的小图标
- 注意：不同分辨率/缩放下的 UI 元素尺寸可能不同，可多准备几份
